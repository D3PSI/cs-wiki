Introduction

The outputs of sequential logic depend on both current and prior input values. Hence, sequential logic has memory. Sequential logic might explicitly remember certain previous inputs, or it might distill the prior inputs into a smaller amount of information called the state of the system. The state of a digital sequential circuit is a set of bits called state variables that contain all the information about the past necessary to explain the future behavior of the circuit.

Latches and Flip-Flops

The fundamental building block of memory is a bistable element, an element with two stable states. The figure shows a simple bistable element consisting of a pair of inverters connected in a loop. The same circuit is redrawn to emphasize the symmetry. The inverters are cross-coupled, meaning that the input of I1 is the output of I2 and vice versa. The circuit has no inputs, but it does have two outputs, QQQ﻿ and Qˉ\bar{Q}Qˉ​﻿. Analyzing this circuit is different from analyzing a combinational circuit because it is cyclic: QQQ﻿ depends on Qˉ\bar{Q}Qˉ​﻿, and Qˉ\bar{Q}Qˉ​﻿ depends on QQQ﻿.

Consider the two cases, QQQ﻿ is 0 or QQQ﻿ is 1. For each case the following consequences result:

  • Case 1: Q=0Q = 0Q=0﻿

    As shown in figure a), I2 receives a FALSE input, QQQ﻿, so it produces a TRUE output on Qˉ\bar{Q}Qˉ​﻿. I1 receives a TRUE input, Qˉ\bar{Q}Qˉ​﻿, so it produces a FALSE output on QQQ﻿. This is consistent with the original assumption that Q=0Q = 0Q=0﻿, so the case is said to be stable.

  • Case 2: Q=1Q = 1Q=1﻿

    As shown in figure b), I2 receives a TRUE input and produces a FALSE output on Qˉ\bar{Q}Qˉ​﻿. I1 receives a FALSE input and produces a TRUE output on QQQ﻿. This is again stable.

Because the cross-coupled inverters have two stable states, Q=0Q = 0Q=0﻿ and Q=1Q = 1Q=1﻿, the circuit is said to be bistable. A subtle point is that the circuit has a third possible state with both outputs approximately halfway between 0 and 1. This is called a metastable state and will be discussed later.

An element with NNN﻿ stable states conveys log⁡2N\log_2 Nlog2​N﻿ bits of information, so a bistable element stores one bit. The state of the cross-coupled inverters is contained in one binary state variable, QQQ﻿. The value of QQQ﻿ tells us everything about the past that is necessary to explain the future behavior of the circuit. Specifically, if Q=0Q = 0Q=0﻿, it will remain 0 forever, and if Q=1Q = 1Q=1﻿, it will remain 1 forever. The circuit does have another node, Qˉ\bar{Q}Qˉ​﻿, but Qˉ\bar{Q}Qˉ​﻿ does not contain any additional information because if QQQ﻿ is known, then Qˉ\bar{Q}Qˉ​﻿ is also known. On the other hand, Qˉ\bar{Q}Qˉ​﻿ is also an acceptable choice for the state variable.

When power is first applied to a sequential circuit, the initial state is unknown and usually unpredictable. it may differ each time the circuit is turned on.

Although the cross-coupled inverters can store a bit of information, they are not practical because the user has no inputs to control the state. However, other bistable elements, such as latches and flip-flops, provide inputs to control the value of the state variable.

SR Latch

One of the simplest sequential circuits is the SR latch, which is composed of two cross-coupled NOR\text{NOR}NOR﻿ gates as shown. The latch has two inputs, SSS﻿ and RRR﻿, and two outputs, QQQ﻿ and Qˉ\bar{Q}Qˉ​﻿. The SR latch is similar to the cross-coupled inverters, but its state can be controlled through the SSS﻿ and RRR﻿ inputs, which set and reset the output QQQ﻿.

A good way to understand an unfamiliar circuit is to work out its truth table, so let’s start there. Recall that a NOR\text{NOR}NOR﻿ gate produces a FALSE output when either input is TRUE. Consider the four possible combinations of RRR﻿ and SSS﻿:

  • Case 1: R=1,S=0R = 1, S = 0R=1,S=0﻿

    N1 sees at least one TRUE input, RRR﻿, so it produces a FALSE output on QQQ﻿. N2 sees both QQQ﻿ and SSS﻿ FALSE, so it produces a TRUE output on Qˉ\bar{Q}Qˉ​﻿.

  • Case 2: R=0,S=1R = 0, S = 1R=0,S=1﻿

    N1 receives inputs of 0 and Qˉ\bar{Q}Qˉ​﻿. Because we don’t yet know Qˉ\bar{Q}Qˉ​﻿, we can’t determine the output QQQ﻿. N2 receives at least one TRUE input, SSS﻿, so it produces a FALSE output on Qˉ\bar{Q}Qˉ​﻿. Now we can revisit N1, knowing that both inputs are FALSE, so the output QQQ﻿ is TRUE.

  • Case 3: R=1,S=1R = 1, S = 1R=1,S=1﻿

    N1 and N2 both see at least one TRUE input (RRR﻿ or SSS﻿), so each produces a FALSE output. Hence QQQ﻿ and Qˉ\bar{Q}Qˉ​﻿ are both FALSE.

  • Case 4: R=0,S=0R = 0, S = 0R=0,S=0﻿

    N1 receives input of 0 and Qˉ\bar{Q}Qˉ​﻿. Because we don’t yet know Qˉ\bar{Q}Qˉ​﻿, we can’t determine the output. N2 receives inputs of 0 and QQQ﻿. Because we don’t yet know QQQ﻿, we can’t determine the output. Now we are stuck. This is reminiscent of the cross-coupled inverters. But we know that QQQ﻿ must either be 0 or 1. So we should be able to solve the problem by checking what happens in each of these subcases.

    • Case 4a: Q=0Q = 0Q=0﻿

      Because SSS﻿ and QQQ﻿ are FALSE, N2 produces a TRUE output on Qˉ\bar{Q}Qˉ​﻿, as shown in a). Now N1 receives one TRUE input, Qˉ\bar{Q}Qˉ​﻿, so its output, QQQ﻿, is FALSE, just as we had assumed.

    • Case 4b: Q=1Q = 1Q=1﻿

      Because QQQ﻿ is TRUE, N2 produces a FALSE output on Qˉ\bar{Q}Qˉ​﻿, as shown in b). Now N1 receives two FALSE inputs, RRR﻿ and Qˉ\bar{Q}Qˉ​﻿, so its output, QQQ﻿, is TRUE, just as we had assumed.

Putting this all together, suppose QQQ﻿ has some known prior value, which we shall call QprevQ_{\text{prev}}Qprev​﻿, before we enter Case 4. QprevQ_{\text{prev}}Qprev​﻿ is either 0 or 1, and represents the state of the system. When RRR﻿ and SSS﻿ are 0, QQQ﻿ will remember this old value, QprevQ_{\text{prev}}Qprev​﻿ and Qˉ\bar{Q}Qˉ​﻿ will be its complement, Qˉprev\bar{Q}_{\text{prev}}Qˉ​prev​﻿. The circuit has memory.

The truth table shown summarizes the four cases. The inputs SSS﻿ and RRR﻿ stand for Set and Reset. To set a bit means to make it TRUE. To reset a bit means to make it FALSE. The outputs, QQQ﻿ and Qˉ\bar{Q}Qˉ​﻿, are normally complementary. When RRR﻿ is asserted, QQQ﻿ is reset to 0 and Qˉ\bar{Q}Qˉ​﻿ does the opposite. When SSS﻿ is asserted, QQQ﻿ is set to 1 and Qˉ\bar{Q}Qˉ​﻿ does the opposite. When neither input is asserted, QQQ﻿ remembers its old value, QprevQ_{\text{prev}}Qprev​﻿. Asserting both SSS﻿and RRR﻿ simultaneously does not make much sense because it means the latch should be set and reset at the same time, which is kind of impossible. The poor circuit has no other choice but to make both outputs 0, which makes no sense.

The SR latch is represented by the symbol shown. Using the symbol is again an application of abstraction and modularity. There are various ways to build an SR latch, such as using different logic gates or transistors. Nevertheless, any circuit element with the relationship specified by the above truth table and the symbol shown is called an SR latch.

Like the cross-coupled inverters, the SR latch is a bistable element with one bit of state stored in QQQ﻿. However, the state can be controlled through the SSS﻿ and RRR﻿ inputs.

D Latch

The SR latch is awkward because it behaves strangely when both inputs are asserted simultaneously. Moreover, the inputs conflate the issues of what and when. Asserting one of the inputs determines not only what the state should be but also when it should change. Circuit design becomes a lot easier when these questions are separated. The D latch in the figure solves these issues. It has two inputs. The data input, DDD﻿, controls what the next state should be. The clock input, CLK\text{CLK}CLK﻿, controls when the state should change.

Again, we can analyze the latch by writing the truth table, given in the figure. We notice that in all cases, Qˉ\bar{Q}Qˉ​﻿ is the complement of QQQ﻿, as would seem logical. The D latch avoids the strange case of simultaneously asserted RRR﻿ and SSS﻿ inputs.

Putting it all together, we see that the clock controls when data flows through the latch. When CLK=1\text{CLK} = 1CLK=1﻿, the latch is transparent. The data at DDD﻿ flows through to QQQ﻿ as if the latch were just a buffer. When CLK=0\text{CLK} = 0CLK=0﻿, the latch is opaque. It blocks the new data from flowing through to QQQ﻿, and QQQ﻿ retains the old value. Hence, the D latch is sometimes called a transparent latch or a level-sensitive latch. The D latch symbol is given in c).

The D latch updates its state continuously while CLK=1\text{CLK} = 1CLK=1﻿.
